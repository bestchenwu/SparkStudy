本章是描述zookeeper方面的学习

解决主从模式的问题，必须解决:
1、主节点崩溃
如果主节点发送错误并失效，系统将无法分配新的任务或重新分配已失败的任务
2、从节点崩溃
如果从节点崩溃，已分配的任务将无法完成
3、通信故障
如果主节点和从节点之间无法进行信息交换，从节点将无法得知新任务分配给它


根据之前描述的这些，我们可以得到以下主-从架构的需求：
主节点选举
这是关键的一步，使得主节点可以给从节点分配任务。
崩溃检测
主节点必须具有检测从节点崩溃或失去连接的能力。
组成员关系管理
主节点必须具有知道哪一个从节点可以执行任务的能力。
元数据管理
主节点和从节点必须具有通过某种可靠的方式来保存分配状态和执
行状态的能力。
